### 2.7 Remediation Plan

#### Fix 1: SQL Injection in qa-test.py

**Before:**
```python
query = f"SELECT * FROM users WHERE id = '{user_id}'"
```

**After:**
```python
query = "SELECT * FROM users WHERE id = ?"
cursor = conn.execute(query, (user_id,))
```

**Why this fix works:** Parameterized queries separate SQL logic from user data, preventing injection.

#### Fix 2: SQL Injection in shop.py

**Before:**
```python
query = "SELECT * FROM orders WHERE user_id = " + str(user_id)
```

**After:**
```python
query = "SELECT * FROM orders WHERE user_id = ?"
cursor = conn.execute(query, (user_id,))
```

**Why this fix works:** Parameterized query prevents user input from altering SQL structure.

#### Fix 3: Remove Hardcoded Admin Password

**Before:**
```python
ADMIN_PASS = "supersecret123"
```

**After:**
```python
ADMIN_PASS = os.environ.get('ADMIN_PASSWORD')
if not ADMIN_PASS:
    raise ValueError("ADMIN_PASSWORD environment variable required")
```

**Why this fix works:** Environment variables keep secrets out of source code.

#### Fix 4: Remove Hardcoded Secret Key

**Before:**
```python
SECRET_KEY = 'hardcoded-secret-123'
```

**After:**
```python
SECRET_KEY = os.environ.get('SECRET_KEY')
if not SECRET_KEY:
    raise ValueError("SECRET_KEY environment variable required")
```

**Why this fix works:** Externalizes secrets to environment configuration.
